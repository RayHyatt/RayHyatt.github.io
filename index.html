需求洞察No.5：案头研究 只言片语藏需求（二）

发表于 2018-06-25 | | 阅读次数:

练就一双甄别信息的慧眼

        现在由于网络信息过于繁杂，随着公关和自媒体的普及，越来越多的信息有存在着潜在的宣传、广告的嫌疑。在信息搜集的过程中，有一双能甄别和选择的慧眼十分重要。

  • 关注事实：注意信息中描述的具体事实性内容，比如具体数据，如何操作等等。把客观事实和文章的态度分开。
  • 偏好的态度进行反向思考：当看到信息有明确的偏好的时候，尝试着去思考一下是否存在不好的可能性。比如说常见的软文可能会只指明一部分数据来说明产品的价值。当你把事实和态度分开的时候，就可以思考一下这样的事实是否还会产生其他的解释。另一个较好的方法是刻意在网上搜索一些负面信息。虽然有些企业自己不会暴露自己的问题，但是他们的竞争对手可不会手软。
  • 把不同来源的信息进行交叉验证：通过多渠道收集到的信息相互之间进行验证，看看他们是否能在同一个逻辑框架下互相解释。

从只言片语中找到用户的价值

从发现需求的角度来说，案头研究主要围绕着四个方面进行信息归纳和整理：

  • 用户：目标用户的分类特征
  • 行业：竞争产品的确认及产品的基本情况
  • 套路：目前产品内的一些基本套路与做法
  • 趋势：产品未来的发展趋势

谁可能是目标用户？

        首先，不管产品或者服务是什么，甚至没有产品或者服务，都没有关系，至少对自己的目标用户多少有个初步的想法，是普通消费者还是企业用户，是男性还是女性，是中老年还是青少年，是国内还是国外。最广泛的情况下，目标用户甚至可以是全人类。
        在明确了有需求的用户以后，需要考虑如何把这些用户进行细分。细分的维度有很多，包括人口学的，行为学的，心理学的等等。
        简单的说，针对用户的内容，需要找到的就是用户和细分。这个环境的主要方法就是搜集一些已经有的分析材料，这些材料包含一些第三方的报告或者一些消费者趋势分析报告，通过这些报告，你可以获得一些业内常用的用户细分标准和各种细分用户的基本特征，然后你需要将你获得的这些分类和特征按照自己的要求进行整合。

这个过程大概可以概括如下：
a) 总结不同的报告的细分标准，把各种细分的维度总结出来，形成一个目标市场上最全的用户细分维度结构。把根据不同报告细分用户的详细特征填充到你总结形成的细分结构中。
b) 思考每一种细分维度被使用的原因是什么？比如，按照年龄来细分的原因是因为年轻人接受新事物比较容易，还是因为中年人的可支配收入比较高，或者是因为老年人比较容易收到现场营销的影响？
c) 基于这两年，再思考是否存在被市场忽略的其他维度，并提出你觉得这些漏掉的维度比较重要的原因。如果能找到一个维度，就说明找到了一个新的切入点，这对未来发现新的需求会有特别的帮助。
d) 根据对不同细分维度被使用原因的分析，将不同的细分维度按照重要性进行排序，尤其是要重点考虑一下漏掉的维度。
e) 根据重要性建立自己的细分框架，形成细分标准和细分用户群

需求洞察No.4：案头研究 只言片语藏需求（一）

发表于 2018-06-22 | | 阅读次数:

        随着互联网的发展，尤其是微博、公众号等自媒体的兴起，关于用户需求的信息已经越来越多，而且也越来越即使的反映各种动态。用户的抱怨，段子手的调侃，商业自媒体的精彩评论，以及各个公司发布的用户研究报告都能在网络上找到，良好的案头研究方法和手段能够极大的加速需求洞察的进程。

隐形信息尤为重要

        案头资料分析并不是一定要搞到内部资料才有价值，还要对搜集的信息进行深度的整理。这是一种个人观点的解读，通过自己的分析把原始的案头信息再加工，经过独立思考，发现潜在机会，或提出有意义的问题。很多人都有一种错误的认知，认为案头资料都是已经分析和整理后的资料，这些内容的信息已经被充分揭露过了，如果油分析的价值，肯定有人已经分析过了，所以不会有更多的剩余价值，只要汇总就可以了。
        事实并不是这样，当年日本情报专家凭借《中国画报》的一张照片推算出大庆油田的具体位置、炼油规模、产油能力等机密数据。
当围绕一个主题把大量的相关信息整合在一起后，信息和信息之间就有可能产生一些化学反应，变成了一种新的知识，进而揭示出一些更生层次的内容——也许是拼凑出完整的需求链条，也许是排除了所有的干扰项，也许是找到了共同的影响因素

如何成为信息搜集达人

  1. 首先通过搜索引擎进行一些关键词搜索，整理一些内容和信息，形成对领域的初步了解。（这种搜索方式找到的信息可能会相对陈旧一些，比较常用，属于比较经典的入门级内容）
  2. 搜索一下是否有相关的行业报告。（在研究调研报告的时候，你会发现一些业内的基本规范、流程、以及行业研究的整体框架）
  3. 搜索一下社交媒体的内容，看看大家都关注什么内容，近期有哪些新消息（比如微博可以直接搜索，微信可以通过搜狗进行搜索）
  4. 查看产品聚合的平台，类似于大众点评网、携程、58同城等平台（这些平台会有目前竞争对手的各类材料，这里需要着重看一下各个竞品的用户评论内容，总结评论中反映出的用户痛点）

  5. 找各种专业论坛，然后进入论坛进行深度搜索，主要搜索与用户及产品领域相关的深度信息（一方面可以为下一步研究作准备，有些内容后续可以直接在论坛中发帖子进行咨询了解，另一方面这种深度搜索往往能够找到一些不错的意见领袖，为进一步的深度沟通作准备。不过由于论坛的帖子较多，搜索信息比较困难，所以可以重点关注四类内容：
    a. 论坛中的精华帖：这些精华帖的内容可能是业内一些约定俗成的内容，甚至是一些新手入门的必备阅读材料，帮助你快速入门。另外关注精华帖的收藏日期，也能知道论坛目前的活跃程度。
    b. 论坛中的大神发帖：好的论坛总有几个大神级的人物在活跃，他们对行业、用户、产品都有着深入的理解，甚至于自己就是个深度用户。
    c. 论坛中的求助帖：尤其是那种一而再，再而三的求助帖，不同的人都在问同样的问题，说明这些问题可能就是用户的痛点，或者是用户的选择和判断标准。
    d. 论坛中阅读和回复量比较高的帖子：这种帖子一般是用来讨论比较热门的话题，常常会有多个观点，可能把一个主题的各个方面的利益都涵盖进去了，从而让你能够迅速的了解各种潜在规则；另一方面，这种帖子记得回复量和阅读量比较高，有可能吸引一些资深人士的加入讨论。

  6. 除此之外，还可以根据关键词搜索一些业内的指数情况，例如百度指数、微博指数、甚至包括Alexa排名等等（这些指数也能在一定程度上帮助你评估各种趋势的发展方向。

        还有一个方法是认识一些虚拟或现实的朋友，他们有一个特点就是喜欢泡网，所以当想研究什么的时候不妨去问问他们，也许会节约你非常多的时间。比如说有什么问题先上“知乎”搜一下，找到几个你觉得靠谱的“回答者”然后再私信咨询一下，一般都会有不错的收获～

需求洞察No.3：用问题去了解用户（二）

发表于 2018-06-17 | | 阅读次数:

根据某项目的核心问题结合六大提纲提出的问题提纲：

一、核心问题

  1. 用户是如何理解家庭医生、健康服务站的？
  2. 家庭医生及健康服务站这种模式的出现对于用户来说意味着什么？

二、基本情况

  1. 姓名、年龄、性别、职业、职位、学历
  2. 平时的时间安排
  3. 目前处于一种什么样的生活状态/环境之中
  4. 对于自己的生活状态/环境希望有什么样的改变
  5. 目前自己的身体健康状况
  6. 家里人的身体健康状况

三、使用情况

  1. 平板电脑、手机、智能手环、及其他穿戴式智能设备的使用情况。
  2. 需要的服务类型、选择的标准和依据、家庭医生的签约和健康服务站的服务购买对于用户来说的意义和价值、希望达到的目的（短期、长期）
  3. 服务场景（家中、家外）、使用服务的频次
  4. 针对每一种场景的描述：过程中的环节描述，环节之间的行为逻辑，过程中的问题，原因是什么？
  5. 过程前后的附加环节（比如：服务后的评价、服务前的预约等）
  6. 服务过程中的情绪以及可能会出现的场景：愤怒、失望、惊吓、惊喜、认可、轻松、高兴。
  7. 最认可的内容、最不认可的内容

四、家庭医生及健康服务站决策

  1. 对家庭医生的理解，在认识中家庭医生是一个什么样的概念？
  2. 对健康服务站的理解，认为健康服务站提供的是一个什么样的环境？
  3. 是否考虑过购买与家庭医生，或者健康服务站的服务类似的产品？如果有产生这种意愿的初衷是什么？
  4. 购买价格和决策
  5. 购买前最担心的问题，看中的家庭医生或是健康服务站的服务的价值是什么？
  6. 哪些情况在体验过后会再次购买？

PS：所有的决策性问题都可以增加一个，作出这样的选择的原因是什么？

        有了问题提纲，他可以指导你在接下来的研究中需要关注的点。接下来就是需要考虑研究方法了，研究方法大致分为二手资料研究和一手资料研究。比如案头研究、访谈与观察、痕迹物分析、日记法等等，根据具体情况我们可能会结合多种方法进行。当确定研究方法后，接下里的就是完善执行计划，如果在你的研究过程中没有提前制定好的计划作为指导，这个过程会浪费你很多时间，甚至丢掉很多关键的环节从而导致研究失败。

执行计划一般分为：
1) 工作阶段和时间计划。列出项目的主要阶段和各项阶段的计划时间。例如：启动阶段、案.头研究阶段（约15%）访谈准备阶段（约30%）用户访谈阶段（约20%）分析阶段（约30%）汇报展示阶段（约5%）
2) 人员配置及分工，如果是一个人就不用说了，如果稍微大一点的项目，可能不同小组执行不同的任务，每个人负责哪个模块，需要输出的结果和标准，都需要一一明了。
3) 样本数量，根据研究项目的具体情况确认自己的样本数量，比如如果你要研究什么会提高产品的使用率，那么邀请一个最爱使用你产品的特殊用户也许就能达到你的目的。不过一般来说建议平均完成10～20（每个类别的画像3-4个）左右的用户研究是一个比较合理的样本数据。
4) 辅助安排。包括研究要使用的工具、场地等等，比如说访谈需要场地、录音笔、照相机等设备。如果让用户完成相应的任务，还需要准备相关道具。
5) 成本预算，研究过程中的活动经费预算。
6) 执行提纲，与问题提纲不一样，是操作问题提纲的细化。需要说明一些操作细节，保证每一个环节完整进行。

需求洞察No.2：用问题去了解用户（一）

发表于 2018-06-12 | | 阅读次数:

本人的用户研究的工作也是初步开始，在开始的时候会遇到很多问题。我该邀请什么样的用户进行调查？我该设计什么样的问题去调查？我应该采用什么样的模式去调查？我该怎么分析我的调查结果？一切的一切都是在一脸懵逼中不断摸索学习，以下是在这个过程中整理的学习笔记和相关感想～如果你也和我拥有一样的疑问，那么不妨看看，也许能帮你解决一些问题呢？

  首先假设我们已经拥有了一个想要调研的问题，那么现在我们是直接拿着这个问题去问用户吗？比如说我要调查一个健康评分系统对用户来说意味着什么。我不可能直接找一个用户拉上来就去问：“诶，你觉得我们设计的这个健康评分系统，如果给你用的话意味着什么呢？”用户显然不能够回答我这个问题，用户能够回答的只是跟他个人相关的一些具体的体验经历，对于更抽象的用户需求要我们自己结合用户提供给我们的信息分析之后给出答案。
  所以说，在用户调研的过程其实是我们收集信息的过程，真正探索需求的过程是我们结合这些信息后续分析的过程，为了方便这个过程我们要获得必要的用户信息。
  在调研之前，为了避免我们收集的信息有差漏，或者是会问出一些用户答不上来的问题，我们需要规划我们的研究提纲，明确需要了解的用户信息。围绕这我们最终想要知道的答案，把我们的问题一层一层分解开来，逐步与用户的经历关联起来。在研究的过程中，我们会随着跟用户互动交流而不断的产生新的想法，那么我们可以把新的想法再一次融入提纲中，并在与新的用户交流时使用。因此我们的提纲设计要保持灵活性和可拓展性。
  虽然每一个项目研究的侧重点有所不同，但是对于用户的研究总体上还是有一定模式可循的。针对用户的问题一般可以分为以下六个方面的内容，我们在设计提纲时可以结合具体情况选择性的挑选想要了解的内容，然后组成问题提纲。


六大提纲

一、基本情况

  用户的基本人口特征、家庭情况、工作情况、兴趣爱好、日常情况等
  说明：为了了解用户基本情况，对用户可以初步打上标签，能够帮我们确认是否用户能够满足需求研究，在后续的分析过程中，也可以作为分类标签进行使用。

二、行为描述

  用户的决策、购买、使用流程如何（包含现在的和期望的），每一个环节有什么更细分的子环节，每一个子环节包含的动作及与其他内容的互动，以及消耗时间。
  说明：了解用户的行为及场景能够更好的与用户接触、引发用户注意，同时能够大最大限度的减少用户的交易成本，促使用户选择。

三、场景及影响因素描述

  用户行为发生的场景是什么样的，不同场景是否对行为有影响，除了场景因素，用户购买和行为还会受到哪些因素影响，不同因素的影响力差异如何。
  说明：了解场景及影响因素能够更好地把握用户需求产生的外因，了解什么驱动用户购买，什么阻止用户行动，了解用户需求产生的真实动机。

四、心理及情绪描述

  用户情绪类型，用户在行为或者互动过程中的每一个环节的情绪变化，用户对产品整体的感觉和认可程度。
  说明：了解用户的情绪能够更好的判断用户在使用过程中的痛点，有助于了解用户真正的需求以及对不同需求的渴望程度。

五、偏好及思维模式描述

  用户思考问题的方式，是信息决策还是凭直觉决策；是自己独立判断还是参考别人经验。在不同的场景下，用户思维模式使不同的。
  说明：用户的思维模式是用户的基础，虽然不同的人有着不同的思维模式，但是对不同细分人群而言，其用户的思维模式还是具有一定的共同性。这种思维模式，一定程度上决定了用户的决策的和判断的机制和方式。

六、生活模式及价值观描述

  主要指用户坚持的原则、用户决策的依据、用户理解风险和价值大小的看法。
  说明：用户的生活模式和价值观决定了用户的选择标准。不同价值观的人对同一件事情的价值判断有可能是不一样的。比如用户不太愿意被打扰，那么说明文档就要做得简单易懂，如果用户需要随时咨询，那么呼叫电话就比较重要。

  了解生活模式、价值观，再加上前面所述的用户思维模式，这样就能够更容易的进入用户角色，能够比较准确的按照用户的思维模式去思考，基于用户的思维模型，能够最大限度的降产品人员本身的思维方式去除掉。成功的摆脱”我认为”的思维模式。

需求洞察No.1：如何提出一个好问题？

发表于 2018-06-11 | | 阅读次数:

问对问题才能找准需求。但是问问题是一门学问，提出好问题就相当于问题解决了一半。提出好问题的过程本身就是一个思考深入的过程。在这里跟大家分享一个我在学习过程中，学到的需求问题修改的模版。

目的性需求的问题改进

No.1首先提出一个问题：
我们应该如何设计积分体系，才能降低用户的流失率？

No.2修改第一步：降问题中涉及的解决方案暂时去掉
因为问题设计出来是帮我们寻找答案的，如果在问题的提出就涉及到解决方案会局限我们的思考。
所以该问题修改为：我们应该如何降低用户流失率？

No.3修改第二步：问题如果是“难题”模式问题，统一改成“疑惑”模式问题。
简单的说，把“怎么办”改为“为什么”
所以该问题修改为：为什么用户流失率这么高？

No.4修改第三部：修改第一步的问题，询问问题的目的
在No.2修改过后的问题上再问一个目的
所以该问题修改为：我们为什么要降低用户流失率？

通过这四部对你的问题进行修改，能够让你从你想要解决的目的出发，分析出现这种情况的现象，跟深入的了解问题之后从更广的方面上去思考这个问题的解决方法。

探索性需求的问题改进

No.1首先提出一个问题：
我们的产品，还能增加什么功能为用户提供什么样的服务？

No.2修改第一步：说明一下产品的核心是什么？
举个例子以理发店为例：产品的核心是为客户理发

No.3修改第二步：在核心产品的基础上问一个“为什么？”
于是产生一个新的问题：客户为什么会理发？——思考并获得答案：为了保持形象和心情。

在这一步中需要保持三个原则让你回答出正确导向的答案。

  • 不要回答客观现象，而要回答用户感受（例如：Q：用户为什么要买面包？A：【错误】因为面包很香【正确】为了能够便捷的满足口腹之欲）
  • 不要讲产品，而是要讲变成的价值和作用（例如【错误】用户想要加湿器【正确】为了保持空气湿润）
  • 价值要体现在对自己的好处上（例如还是上面那个问题回答成：想让皮肤保持湿润，让皮肤变得更有弹性）

No.4修改第三步：在第二步的基础上问一个“还有什么？”
于是产生一个新的问题：还有什么服务或产品可以保持形象和心情？

No.5修改第四步：仍然在第二步的基础上问一个“意味着什么？”
于是产生一个新的问题：保持形象和心情意味着什么？

总结问题

  1. 还有什么服务或产品可以保持形象和心情？
  2. 保持形象和心情意味着什么？

第一个问题的提出是为了扩展研究范围，考虑间接竞争者。
第二个问题的提出是为了思考价值和意义。之所以问意味着什么？而不是问为什么？核心目的是希望让用户的回答能够更发散，能够更聚焦于意义，能够阐释出更多的关联关系，而不仅仅是因果关系。

写在最后的话

一个好的问题应该具有的特点：

  • 好的问题能够帮助你发现新的认知需求的角度
  • 好的问题能够激发去探索更深层次的原因和意义
  • 好的问题能够英法范围和层次更广的思考

希望更大家分享的方法能够帮助大家在学习，生活和工作中更好的处理手上的问题～共勉！
